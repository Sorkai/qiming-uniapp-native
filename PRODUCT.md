# Product

## Register

product

## Users

Teachers, students, and school administrators use this web product during active teaching and learning workflows. Teachers need to inspect courses, resources, learning profiles, assessments, and governance signals without losing context. Students need clear guidance, readable feedback, and course-aware AI assistance. Administrators need trustworthy monitoring surfaces for risk, content quality, and system traceability.

## Product Purpose

Qiming Smart Education is an AI-assisted education platform for course operations, learning support, assessment, resource generation, and governance. The AI App should not feel like an isolated chat demo. It should make the learning loop legible: course context, student profile, learning path, generated resources, evaluation evidence, and governance state should reinforce each other.

## Brand Personality

Calm, credible, and task-focused. The interface should feel like a serious education SaaS product with intelligent assistance in the background, not a decorative AI showcase. Confidence should come from clarity, hierarchy, and reliable states rather than visual effects.

## Anti-references

Avoid generic AI app tropes: purple-pink gradients, gradient text, decorative glassmorphism, glowing borders, oversized empty-state animations, repeated rounded cards, template dashboard tiles, and copy that sounds like an AI demo. Avoid making every page look like a separate prototype. Avoid visual novelty that competes with teaching data or workflow completion.

## Design Principles

1. Treat AI as workflow infrastructure, not spectacle.
2. Preserve context across course, student, profile, path, resource, assessment, and governance views.
3. Prefer earned familiarity over decorative surprise.
4. Use restrained surfaces, stable hierarchy, and consistent component vocabulary.
5. Empty, loading, and error states should explain recovery and next action.

## Accessibility & Inclusion

Target WCAG AA contrast and keyboard accessibility. Interactive targets should be comfortably clickable, focus states visible, and state changes understandable without color alone. Motion should be purposeful, short, and respectful of reduced-motion preferences. Text should remain readable for Chinese and English content, long course names, and mixed technical labels.
