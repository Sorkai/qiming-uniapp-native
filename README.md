# 启明智教

启明智教是一套面向学校、教师与学生的 AI 教学服务平台前端工程。项目围绕课程建设、资源学习、题库组卷、作业考试、学情分析、竞赛训练、虚拟实验与 AI 学习助手等场景，提供可持续演进的教学业务控制台与学习端体验。

本仓库承载启明智教 Web 端的核心界面、路由、权限、业务模块、静态资源与前端接口约定，配套文档位于 `doc/` 目录。

## 项目定位

- **教学业务中台**：支持课程、分类、教案、资源、作业、考试、讨论、通知等教学管理流程。
- **学生学习空间**：提供课程学习、测验作答、错题复盘、学习画像、学习计划、资料研读等学习体验。
- **AI 能力入口**：承载互动答疑、教学资源生成、资料研读、学习路径、学情评估、治理看板和常规任务等智能体能力。
- **综合实践场景**：覆盖编程竞赛、知识竞赛、作文比赛、虚拟实验、虚拟校园、云盘与在线资料管理。
- **工程化前端底座**：基于 Vue 3、Vite、TypeScript、Element Plus、Pinia、Vue Router 等技术栈，面向长期迭代与多人协作。

## 智能学习闭环

启明智教没有把 AI 助手设计成孤立的问答窗口，而是围绕学生在具体课程中的持续学习过程，组织成可追踪、可调整、可治理的智能学习闭环：课程资料和学习记录提供上下文，画像与路径决定个性化策略，资源生成与即时辅导承接学习动作，评估与治理再回流到后续计划。

| 环节             | 设计重点                                                                                                                                     | 当前落地                                                               |
| ---------------- | -------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------- |
| 课程知识底座     | 以专业课程为单元沉淀章节、资料、作业、考试、视频、讨论和文档集，为 AI 推理、问答和资源生成提供稳定上下文                                     | 课程中心、课程详情、资料研读、题库与作业考试                           |
| 动态学习画像     | 结合对话、学习进度、作答记录、错题、资源使用和教师校准，持续刻画知识基础、薄弱点、学习目标、偏好、节奏与风险信号                             | 学习画像、学生档案、画像刷新、画像纠偏与 AI Inspector                  |
| 协作式资源生成   | 将画像分析、路径规划、讲解生成、练习生成、资源审校拆分为可追踪步骤，生成讲解文档、PPT、题库、思维导图、拓展阅读、视频/动画脚本与代码实操材料 | 教学资源工作台、AI PPT、AI 动画、题库、资源任务、执行 Trace 与资源审核 |
| 个性化路径规划   | 基于画像、课程目标、学习进度和评估结果生成阶段化学习路径，支持节点推进、版本历史、动态重规划和资源推送                                       | 学习计划、路径节点、路径版本、路径推送任务                             |
| 即时辅导与研读   | 在课程和资料上下文中答疑，支持文字解释、图文材料、引用来源、安全状态展示、流式对话和 PDF/文档研读                                            | 互动答疑、Agent PDF 工作台、会话 Trace 与安全状态面板                  |
| 学习效果评估     | 汇总练习测试、作业考试、资源反馈和路径完成情况，形成强弱项判断，并驱动画像更新与路径调整                                                     | 测验评估、学情分析、错题复盘、考试统计与课程成绩分析                   |
| 体验与响应机制   | 面向 AI 场景提供清晰的任务状态、生成进度、Markdown 渲染、多模态内容卡片和长任务反馈，减少等待过程中的不确定感                                | 流式对话、资源生成进度、内容卡片、任务状态与降级提示                   |
| 安全治理与可观测 | 对生成内容、资源状态、会话安全、引用来源和执行链路进行审核与追踪，降低黑盒生成和错误传播风险                                                 | 治理看板、资源治理、内容审核、敏感词管理、任务/会话 Trace 查询         |
| 工程与文档沉淀   | 将智能体设计、接口契约、部署流程、测试说明和演示材料沉淀为可复用资产，便于后续课程扩展、联调验收和团队协作                                   | `doc/` 文档体系、Docker、CI/CD、接口契约与部署说明                     |

## 环境要求

- Node.js: `^18.18.0 || ^20.9.0 || >=22.0.0`
- pnpm: `>=9`
- 推荐使用 Node.js 20 LTS 作为本地开发与 Docker 构建环境

项目通过 `preinstall` 强制使用 `pnpm`，不要使用 `npm install` 或 `yarn install` 安装依赖。

## 快速开始

```bash
# 安装依赖
pnpm install

# 启动开发服务
pnpm dev
```

默认开发地址：

```text
http://localhost:8848
```

开发环境下，前端请求默认走 `/api` 前缀，并由 Vite 代理到 `.env.development` 中的 `VITE_PROXY_TARGET`。

## 环境变量

项目使用 Vite 多环境配置：

| 文件               | 说明                                                           |
| ------------------ | -------------------------------------------------------------- |
| `.env`             | 所有环境共享配置，包括本地端口、首页开关、多租户开关、代理目标 |
| `.env.development` | 开发环境配置，默认 API 为 `/api`                               |
| `.env.staging`     | 预发布环境配置                                                 |
| `.env.production`  | 生产环境配置，默认 API 指向线上服务                            |

关键变量：

| 变量                       | 说明                | 示例                             |
| -------------------------- | ------------------- | -------------------------------- |
| `VITE_PORT`                | 本地开发端口        | `8848`                           |
| `VITE_PUBLIC_PATH`         | 静态资源基础路径    | `/`                              |
| `VITE_API_URL`             | 前端请求的 API 地址 | `/api`                           |
| `VITE_PROXY_TARGET`        | 开发代理目标服务    | `https://aiedu-api.intelledu.cn` |
| `VITE_ROUTER_HISTORY`      | 路由模式            | `"hash"`                         |
| `VITE_ENABLE_TENANT`       | 是否启用多租户      | `true`                           |
| `VITE_MOCK_SCOPE`          | Mock 范围           | `exam-paper-only`                |
| `VITE_DOCMEE_API_KEY`      | AI PPT 能力所需密钥 | 按环境配置                       |
| `VITE_DOCMEE_CONTAINER_ID` | AI PPT 容器 ID      | `aippt-container`                |

敏感配置不应直接提交到公共仓库。需要对外发布时，请使用部署平台或运行时配置注入对应密钥。

## 构建与部署

生产构建：

```bash
pnpm build
```

构建产物位于：

```text
dist/
```

部署方式：

- **Nginx 静态部署**：将 `dist/` 内容部署到 Web 根目录。
- **Docker 部署**：使用根目录 `Dockerfile` 构建前端镜像。
- **CNB CI/CD**：参考 `doc/CI-CD-CNB.md` 配置自动化构建与制品发布。

更完整的部署说明见 [前端部署文档](./doc/FRONTEND-DEPLOYMENT.md)。

## 分支策略

| 分支     | 定位              | 说明                                                 |
| -------- | ----------------- | ---------------------------------------------------- |
| `main`   | 生产稳定分支      | 当前线上版本，以可交付、可回滚为优先级               |
| `dev`    | 测试集成分支      | 用于联调、测试、验收前的集成验证                     |
| `derive` | 修复验证分支      | 用于验证 `dev` 或 `main` 的 bug fix 与小范围风险修复 |
| `agent`  | AI 智能体演进分支 | 用于持续演进 AI 助手、智能体、虚拟人、工具调用等能力 |

推荐流程：

1. 新功能优先进入 `dev` 进行集成与验收。
2. 生产问题先在 `derive` 验证，再合入 `dev` 与 `main`。
3. `agent` 分支承载 AI 能力长期演进，公共修复按需从 `main` 同步。
4. `agent` 专属功能不直接回流 `main`，待能力稳定后再规划正式上线切换。

## 大文件与静态资源约定

GitHub 单文件硬限制为 100MB。虚拟人、TTS、模型、训练数据等大体积资源不应直接提交到 Git。

当前已忽略的本地资源示例：

```text
public/virtual-people/tts/sherpa-baker/sherpa-onnx-wasm-main-tts.data
```

如需在生产环境使用此类资源，请通过对象存储、制品库、CDN、部署脚本或 Git LFS 方案进行分发。

## 代码质量约定

- 使用 TypeScript 编写业务逻辑，Vue 单文件组件保持职责清晰。
- 路由、权限、菜单配置统一维护在 `src/router/`。
- 公共状态使用 `src/store/` 中的 Pinia 模块。
- 业务接口与后端契约变更需同步更新 `doc/` 中的对应说明。
- 提交前建议执行：

```bash
pnpm typecheck
pnpm lint
pnpm build
```

## 项目说明

启明智教不是模板仓库，也不是通用后台管理系统展示项目。它是围绕真实教学业务持续演进的 AI 教育产品工程。后续迭代应优先保持业务命名清晰、模块边界稳定、接口文档同步、运行部署可复现。
